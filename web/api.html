Comece em 1 minuto (Quickstart)

Escolha a sua tecnologia preferida e execute a chamada para buscar as cidades disponíveis e consultar um ponto de alcançabilidade:

# 1. Listar todas as cidades processadas no banco
curl -X GET "http://localhost:3000/api/v1/cidades"

# 2. Consultar o tempo de alcançabilidade no centro de Águas de São Pedro
curl -X GET "http://localhost:3000/api/v1/alcancabilidade?cidade_id=21&lat=-22.609922&lon=-47.864508"
// Consultar cidades cadastradas e seus índices de alcançabilidade
async function testarAPI() {
  const resCidades = await fetch('http://localhost:3000/api/v1/cidades');
  const cidades = await resCidades.json();
  console.log('Cidades:', cidades);

  if (cidades.length > 0) {
    const resPonto = await fetch(`http://localhost:3000/api/v1/alcancabilidade?cidade_id=${cidades[0].id}&lat=-22.6099&lon=-47.8645`);
    const dados = await resPonto.json();
    console.log('Dados do Ponto Clicado:', dados);
  }
}

Conceitos em 30 segundos

Cidade Representa a área territorial urbana delimitada no OpenStreetMap cujos serviços e nós foram baixados e processados.
Nó (Node) Uma interseção viária real (esquina ou junção de vias de pedestres) gerada a partir da malha viária do OSM.
Categoria Tipos de serviços essenciais. Chaves válidas: saude, farmacia, educacao, mercado, transporte, banco, combustivel, lazer, ponto_onibus, posto_saude, creche, padaria.
Índice Métrica de 0 a 100 calculada sobre a proporção de categorias que podem ser acessadas a pé no limiar de tempo de 15 minutos.

Método Caminho Descrição Parâmetros da Query
GET /saude Health check da API e do banco de dados Nenhum
GET /cidades Lista as cidades processadas e seus índices consolidados Nenhum
GET /cidades/{id} Detalhes e índices de alcançabilidade de uma cidade Nenhum
GET /cidades/{id}/servicos Retorna pontos de serviços mapeados (GeoJSON) categoria (opcional)
GET /cidades/{id}/isocronas Gera o polígono da isócrona de acesso para a categoria categoria (obrigatório), minutos (default 15)
GET /cidades/{id}/mapa Retorna nós amostrais da cidade para mapa de calor categoria (obrigatório), max (default 3000), velocidade, categorias
GET /cidades/{id}/moreno Diagnóstico de Moreno dinâmico e flexível (Fase 10) categorias (CSV opcional), velocidade (opcional)
GET /alcancabilidade Calcula tempos de alcançabilidade a partir de lat/lon cidade_id (obrigatório), lat (obrigatório), lon (obrigatório), velocidade
GET /rota Traça rota real a pé pela malha viária viária (GeoJSON) cidade_id (obrigatório), de (nó inicial), para (nó final), velocidade
GET /comparar Compara os índices consolidados de até 5 cidades cidades (CSV de IDs)
POST /processamentos Dispara o processamento de um novo município no OSM Nenhum (Envio do body JSON com consulta_osm)
GET /processamentos/atual Retorna o progresso do processamento em andamento Nenhum

Receita Completa: Desenhar Isócronas com Leaflet

Exemplo estático completo que consome a API do projeto local e plota a isócrona no mapa Leaflet. Copie, salve como mapa.html e abra no navegador:

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" />
  <script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
  <style>#map { height: 400px; width: 100%; border-radius: 8px; }</style>
</head>
<body>
  <h3>Visualizador de Isocronas a pe</h3>
  <div id="map"></div>
  <script>
    const map = L.map('map').setView([-22.6099, -47.8645], 14);
    L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png').addTo(map);

    // Consome o polígono da isócrona de lazer para a cidade de Águas de São Pedro (ID: 21)
    fetch('http://localhost:3000/api/v1/cidades/21/isocronas?categoria=lazer&minutos=15')
      .then(res => res.json())
      .then(geojson => {
        L.geoJSON(geojson, {
          style: { color: '#2ecc71', fillColor: '#2ecc71', fillOpacity: 0.2 }
        }).addTo(map);
      })
      .catch(err => console.error("Erro ao carregar isocrona:", err));
  </script>
</body>
</html>

Adicionar uma Cidade via API

Para adicionar novas cidades programaticamente à plataforma, siga o fluxo de execução abaixo:

POST /api/v1/processamentos (com {"consulta_osm": "Guarujá, SP"}) │ ▼ Retorna 202 Accepted {"id": "job-uuid"} ┌──────────────────────────────────────────────┐ │ Loop de Polling a cada 2s │ │ GET /api/v1/processamentos/atual │ │ ├── Se job.status === "rodando" -> continua│ │ └── Se job.status === "concluido" -> fim │ └──────────────────────────────────────────────┘ │ ▼ Cidade disponível em GET /api/v1/cidades 
// Exemplo de script de processamento e polling
async function processarCidade(consulta) {
  const postRes = await fetch('http://localhost:3000/api/v1/processamentos', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ consulta_osm: consulta })
  });
  
  const postData = await postRes.json();
  if (postRes.status === 200) {
    console.log("Cidade já existe! ID:", postData.cidade_id);
    return;
  }
  
  const jobId = postData.id;
  console.log("Processamento iniciado. Job ID:", jobId);

  const timer = setInterval(async () => {
    const pollRes = await fetch('http://localhost:3000/api/v1/processamentos/atual');
    const pollData = await pollRes.json();
    const job = pollData.job;

    if (!job || job.id !== jobId) {
      clearInterval(timer);
      console.log("Job cancelado ou modificado no servidor.");
      return;
    }

    console.log(`Progresso: ${job.pct}% | ${job.msg}`);

    if (job.status === 'concluido') {
      clearInterval(timer);
      console.log("Concluído! Nova Cidade ID:", job.cidadeId);
    } else if (job.status === 'erro') {
      clearInterval(timer);
      console.log("Erro no processamento:", job.msg);
    }
  }, 2000);
}

Boas Práticas e Limites

  • Limitação de Requisições (Rate Limit): A API restringe o acesso a no máximo 100 requisições por IP a cada 15 minutos. Endpoints de progresso em tempo real (/processamentos/atual) e de listagem geral de cidades estão isentos dessas regras para evitar quedas no monitoramento.
  • CORS Aberto: Cabeçalhos CORS estão configurados para permitir leitura a partir de qualquer domínio, facilitando integrações baseadas inteiramente no lado do cliente.
  • Erros e Respostas: Chamadas com erro retornam objetos JSON estruturados no formato: {"erro": "Mensagem detalhada", "codigo": 404}.
